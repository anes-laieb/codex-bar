# Codex Bar 2.1.11

Codex Bar 2.1.11 turns the menu-bar update notice into a complete, verified in-app
update. Clicking **Update Available** now downloads and installs the matching release,
shows progress, and relaunches the app—without sending you to a browser during the
normal update path.

## What changed

- **One-click installation:** the existing update menu item starts the update directly.
- **Visible progress:** a focused window shows the download and verification stages;
  downloads can be cancelled before installation begins.
- **Strict release selection:** only the ZIP whose version and CPU architecture match the
  GitHub release and the running Mac is accepted.
- **Layered verification:** Codex Bar checks the asset size and GitHub-provided SHA-256
  digest, then validates the app's bundle identifier, version, executable architecture,
  and code signature.
- **Safe replacement:** the verified bundle is staged on the installation volume, the
  current app is retained as a rollback copy, and Codex Bar relaunches automatically.
- **Clear recovery:** a failed download or validation leaves the installed version
  untouched. If the installation folder is not writable, the app explains the issue and
  offers the release page as a manual fallback.

## First update

Codex Bar 2.1.10 and earlier do not contain the installer, so they still open the GitHub
release page. Install 2.1.11 manually once. Updates released after 2.1.11 can then use the
direct in-app flow.

## Download

| Asset | Platform | Architecture | Minimum OS |
| --- | --- | --- | --- |
| `Codex-Bar-2.1.11-macOS-arm64.dmg` | macOS installer | Apple silicon (`arm64`) | macOS 13 |
| `Codex-Bar-2.1.11-macOS-arm64.dmg.sha256` | DMG checksum | SHA-256 | — |
| `Codex-Bar-2.1.11-macOS-arm64.zip` | macOS | Apple silicon (`arm64`) | macOS 13 |
| `Codex-Bar-2.1.11-macOS-arm64.zip.sha256` | ZIP checksum | SHA-256 | — |

Verify each package with its matching checksum from the same directory:

```sh
shasum -a 256 -c Codex-Bar-2.1.11-macOS-arm64.zip.sha256
shasum -a 256 -c Codex-Bar-2.1.11-macOS-arm64.dmg.sha256
```

For the standard macOS installation experience, open the DMG and drag **Codex Bar.app**
onto the **Applications** shortcut. The ZIP is the verified payload used by the in-app
updater.

The build is ad-hoc signed and is **not Apple-notarized** because the project does not
currently have a Developer ID signing identity. On first launch, use
**Control-click → Open** if macOS asks you to confirm the application.

## Privacy and network behavior

Automatic checks request public release metadata from GitHub. The ZIP is downloaded only
after the user clicks **Update Available**. No Codex session content or local preference
data is sent during either request.

## Verification

The release is produced by `scripts/package-release.sh`, which performs a clean Swift
build, validates `Info.plist`, verifies the complete application signature, checks the
bundled license, notice, and status assets, validates ZIP and DMG integrity, and emits
separate SHA-256 checksums.
