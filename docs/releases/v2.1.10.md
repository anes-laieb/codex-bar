# Codex Bar 2.1.10

Codex Bar 2.1.10 is a documentation and release-polish update focused on showing the
product's defining feature—the live macOS menu bar—clearly and accurately.

## What changed

- Added a detailed README walkthrough for idle, working, attention, and open-menu states.
- Shows the live timer/activity layout, sessions, weekly usage, New Chat, recent activity,
  hidden sessions, Quick Preferences, update checks, and app controls.
- Keeps the existing real sessions-window preview and clearly distinguishes screenshots,
  deterministic UI walkthroughs, and decorative artwork.
- Added icon-led project links and section headings for faster README navigation.
- Consolidated media documentation into one capture guide and removed the redundant
  nested media README.
- Replaced the visible Apache appendix placeholder with the actual project copyright:
  `Copyright 2026 the Codex Bar contributors`.

## Application behavior

This patch does not change session discovery, icon appearances, rollout parsing,
notifications, preferences, privacy, or network behavior. The bundle version is updated
so the downloadable artifact contains the corrected license and matches the published
documentation.

## Download

| Asset | Platform | Architecture | Minimum OS |
| --- | --- | --- | --- |
| `Codex-Bar-2.1.10-macOS-arm64.zip` | macOS | Apple silicon (`arm64`) | macOS 13 |
| `Codex-Bar-2.1.10-macOS-arm64.zip.sha256` | Checksum | SHA-256 | — |

Verify both downloaded files from the same directory:

```sh
shasum -a 256 -c Codex-Bar-2.1.10-macOS-arm64.zip.sha256
```

The build is ad-hoc signed and is **not Apple-notarized** because the project does not
currently have a Developer ID signing identity. On first launch, use
**Control-click → Open** if macOS asks you to confirm the application.

## Verification

The release is produced by `scripts/package-release.sh`, which performs a clean Swift
build, validates `Info.plist`, verifies the complete application signature, checks the
bundled license, notice, and status assets, validates ZIP integrity, and emits a SHA-256
checksum.
