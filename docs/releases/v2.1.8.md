# Codex Bar 2.1.8

Codex Bar 2.1.8 turns the menu-bar indicator into a complete multi-session control center
for Codex Desktop and Codex CLI. It brings every active task into one native macOS window,
opens exact conversations in one click, and gives completion and attention events their
own notification and sound behavior.

## Highlights

- Discover and follow multiple currently open Codex sessions.
- Open an exact task through its native `codex://threads/{id}` deep link.
- Pin, mute, hide, and restore individual sessions.
- Receive actionable completion and question/approval notifications.
- Configure separate sounds, previews, and scheduled quiet hours.
- Review recent activity and locally derived weekly usage.
- Choose System or Sky Blue status artwork with a 121-frame working animation.
- Control timer visibility, Dock presence, Launch at Login, and update checks.

## What changed

### Sessions and navigation

Codex Bar now joins Codex's local thread index to rollout files held open by Codex,
maintains one tracker per active session, and sorts sessions by attention, working, and
idle priority. Project and session names, model, effort, elapsed time, and task actions
are available from the redesigned window and menu.

### Alerts and preferences

Completion and question/approval events now have independent sounds and native
notifications. Notifications can open the exact task or mute that session. Quiet hours
can cross midnight, and preferences now include timer, sounds, appearance, Dock presence,
Launch at Login, and release checks.

### Artwork and interface

The release replaces the former sparkle presentation with supplied Codex artwork, an
animated working state, a Sky Blue appearance, a redesigned sessions window, recent
activity, weekly usage, and a richer live menu.

### Documentation and project policy

The repository now documents architecture, development, releases, privacy, security,
support, conduct, and its issues-only contribution policy. Bug and feature-request issue
templates are included.

## Privacy and network behavior

Session discovery, rollout parsing, recent activity, and usage calculation happen on the
Mac. Codex Bar does not modify Codex's database, logs, or configuration. Its only direct
network request is an enabled or manually triggered check of public GitHub release
metadata; it never installs an update automatically.

## Verification

The release was produced with `scripts/package-release.sh`, which performs a clean Swift
build, validates `Info.plist`, verifies the complete app bundle's code signature, checks
the packaged license, notice, and status assets, validates the ZIP archive, and emits a
SHA-256 checksum.

The published release asset records its exact source commit in the GitHub release notes.
