# Codex Bar 2.1.9

Codex Bar 2.1.9 corrects and expands the menu-bar icon appearance model. The app now
offers three explicit choices—Default, System, and Sky Blue—and the README shows the
actual artwork used by each option.

## What changed

- **Default** uses the branded dark-navy, full-color idle artwork and the adaptive
  template animation while working.
- **System** uses the white/black template SVG and automatically follows macOS menu-bar
  contrast. It remains the fresh-install default.
- **Sky Blue** uses the color-forward idle artwork and sky-blue working animation.
- Both the full Preferences window and menu-bar Quick Preferences expose all three modes.
- Existing System, Sky Blue, and legacy `colored` saved values migrate without a reset.

Verify the downloaded archive and checksum from the same directory:

```sh
shasum -a 256 -c Codex-Bar-2.1.9-macOS-arm64.zip.sha256
```

This build is ad-hoc signed and is **not Apple-notarized** because the project does not
currently have a Developer ID signing identity. On first launch, use
**Control-click → Open** if macOS asks you to confirm the application.

## Compatibility and privacy

This patch does not change session discovery, rollout parsing, notifications, local
history, or network behavior. Preferences remain stored locally in macOS user defaults.
Codex Bar still makes no changes to Codex's database, logs, or configuration.

## Verification

The release is produced by `scripts/package-release.sh`, which performs a clean Swift
build, validates `Info.plist`, verifies the complete application signature, checks the
bundled license, notice, and status assets, validates ZIP integrity, and emits a SHA-256
checksum.
